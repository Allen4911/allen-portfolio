4:T958,# Migrating React Projects to TypeScript

## Why Migrate Incrementally?

A big-bang migration of a large codebase almost always causes regressions. TypeScript's `allowJs` flag lets you migrate file-by-file, keeping production stable throughout.

## Initial tsconfig Setup

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "checkJs": false,
    "skipLibCheck": true,
    "strict": false,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "paths": { "@/*": ["./src/*"] }
  }
}
```

Start with `strict: false` and `checkJs: false` — tighten these as you go.

## Migration Strategy

### Step 1: Rename entry points first

Rename `_app.js` → `_app.tsx`, `_document.js` → `_document.tsx`. These are the highest-value files.

### Step 2: Migrate leaf components

Start with components that have no imports from other JS files. Pure presentational components are easiest.

```tsx
// Before (JS)
export function Button({ label, onClick, disabled }) {
  return <button onClick={onClick} disabled={disabled}>{label}</button>;
}

// After (TS)
interface ButtonProps {
  label: string;
  onClick: () => void;
  disabled?: boolean;
}

export function Button({ label, onClick, disabled }: ButtonProps) {
  return <button onClick={onClick} disabled={disabled}>{label}</button>;
}
```

### Step 3: Type data files

```ts
// src/types/project.ts
export interface Project {
  id: string;
  title: string;
  category: string;
  github: string | null;
  demo: string | null;
  featured: boolean;
}
```

## Common Patterns

### Event handlers
```tsx
const handleClick = (e: React.MouseEvent<HTMLButtonElement>) => { ... };
const handleChange = (e: React.ChangeEvent<HTMLInputElement>) => { ... };
```

### useRef with DOM elements
```tsx
const ref = useRef<HTMLDivElement>(null);
```

### Next.js page props
```tsx
import type { NextPage, GetStaticProps } from 'next';
const Page: NextPage<Props> = ({ data }) => { ... };
```

## Enabling Strict Mode Gradually

Once all files are `.tsx`, enable stricter checks one at a time:
1. `"noImplicitAny": true`
2. `"strictNullChecks": true`
3. `"strict": true`
