## 개요

위성 통신 시스템은 위성이 운용되는 궤도 고도에 따라 LEO, MEO, GEO로 분류됩니다. 각 궤도는 물리 법칙에 의해 결정되는 고유한 특성을 가지며, 이는 통신 시스템 설계 전반에 걸쳐 근본적인 제약과 장점을 동시에 제공합니다.

---

## 궤도별 기본 파라미터

```
                  ┌────────────────────────────────────────┐
   GEO (35,786 km)│                   ★ GEO               │ 고정점 유지
                  │                                        │
                  │                                        │
   MEO (~10,000 km│           ★ MEO 대역                  │ 수 시간 주기
                  │         (GPS: 20,200 km)               │
                  │                                        │
   LEO (~550 km)  │   ★ LEO 군집                          │ ~90분 주기
                  └────────────────────────────────────────┘
                              지구 표면
```

| 항목 | LEO | MEO | GEO |
|------|-----|-----|-----|
| 고도 | 200 ~ 2,000 km | 2,000 ~ 35,786 km | 35,786 km (고정) |
| 공전 주기 | 90 ~ 127분 | 2 ~ 24시간 | 24시간 (정지) |
| 위성 속도 | ~7.5 km/s | ~3.9 km/s | ~3.1 km/s |
| 편도 전파 지연 | 1 ~ 10 ms | 10 ~ 100 ms | ~270 ms |
| 왕복 RTT | 2 ~ 20 ms | 20 ~ 200 ms | ~540 ms |
| 단일 위성 커버리지 | ~1,000 km 반경 | ~5,000 km 반경 | ~6,000 km 반경 |
| 글로벌 커버리지에 필요한 위성 수 | 수백 ~ 수천 기 | 수십 기 | 3 기 (이론상) |
| 자유공간 경로손실 (Ka-band) | ~170 dB | ~185 dB | ~210 dB |

## 1. LEO (Low Earth Orbit)

### 궤도 역학

```
케플러 제3법칙:  T² = (4π²/GM) × a³

고도 550 km 기준:
  반장축 a = 6,371 + 550 = 6,921 km
  공전 주기 T ≈ 95.5분
  궤도 속도 v = 2πa/T ≈ 7.59 km/s
```

### 자유공간 경로손실 (FSPL)

```python
import math

def fspl_dB(distance_km, freq_GHz):
    c = 3e8
    f = freq_GHz * 1e9
    d = distance_km * 1e3
    lam = c / f
    return 20 * math.log10(4 * math.pi * d / lam)

# Ka-band (26.5 GHz), 고도별 경로손실
for alt in [550, 1200, 2000]:
    loss = fspl_dB(alt, 26.5)
    print(f"고도 {alt:5d} km: {loss:.1f} dB")

# 출력:
# 고도   550 km: 170.3 dB
# 고도  1200 km: 176.0 dB
# 고도  2000 km: 179.5 dB
```

### LEO 도플러 특성

LEO 위성은 빠른 이동으로 인해 가장 큰 도플러 편이가 발생합니다.

```
최대 도플러 편이 (Ka-band 26.5 GHz, LEO 550 km, 수직 통과):

  Δf_max = f_c × (v / c) = 26.5×10⁹ × (7590 / 3×10⁸) ≈ 670 kHz

  ← 위성 접근 시: +670 kHz
  → 위성 이탈 시: -670 kHz
  수직 통과 시간: 약 10분 (仰角 10° 이상 기준)
```

**주파수 편이 보정 방법:**
- UE 사전 보정 (Pre-compensation): GNSS 위치 + 위성 궤도력(Ephemeris) 활용
- 네트워크 기반 보정: 지상국에서 공통 도플러 제거 후 링크 유지
- OFDM 서브캐리어 간격 확대: 도플러 대비 파일럿 설계

### LEO 핸드오버

```
LEO 위성 가시 시간: 약 5 ~ 15분 (仰角 10° 기준)

핸드오버 전략:

  [위성 A] ──── 가시 시간 ────►[위성 B]
      │                             │
      │   ← 핸드오버 결정 구간 →    │
      └──── 소프트 핸드오버 ────────┘

  - 위성간 핸드오버 빈도: ~6 회/시간
  - ISL(위성 간 링크) 활용 시 지상국 거치지 않고 연속성 유지
  - Make-before-break 방식 필수
```

### LEO 대표 시스템

| 시스템 | 운영사 | 위성 수 | 고도 | 특징 |
|--------|--------|---------|------|------|
| Starlink | SpaceX | ~6,000기+ | 550 km | ISL 탑재, 민간/군 겸용 |
| OneWeb | Eutelsat | ~648기 | 1,200 km | 광대역 브로드밴드 |
| Kuiper | Amazon | 3,236기 계획 | 590~630 km | AWS 연동 |
| Telesat Lightspeed | Telesat | 298기 | 1,000 km | 엔터프라이즈 특화 |
| ANASIS-계획 | 한국 (군) | — | LEO 검토 중 | 전술 통신 |

## 2. MEO (Medium Earth Orbit)

### 궤도 역학

```
대표 고도: 10,000 ~ 20,200 km

GPS (20,200 km):
  공전 주기 ≈ 12시간
  궤도 속도 ≈ 3.87 km/s

O3b MEO (8,063 km):
  공전 주기 ≈ 4.8시간
  RTT ≈ 54 ms
```

### MEO 주요 특성

```
장점:
  ✓ LEO 대비 넓은 커버리지 (위성 수 적게 필요)
  ✓ GEO 대비 낮은 전파 지연
  ✓ 반알렌대(Van Allen Belt) 일부 회피 가능
  ✓ 지상 추적 장비 부담 완화

단점:
  ✗ 반알렌대 일부 구간 통과 → 방사선 환경 심각
  ✗ LEO 대비 여전히 핸드오버 필요
  ✗ GEO 대비 더 많은 위성 필요
```

### 방사선 환경 (Van Allen Belt)

```
고도별 방사선 강도:

    0         5,000      15,000      35,786 km
    │─────────────────────────────────────────►
    │                 ████████
    │           ████████████████               ← 내부 Van Allen 대
    │     ██████████████████████████████
    │ ████████████████████████████████████████  ← 외부 Van Allen 대
    └─────────────────────────────────────────
    낮음   높음         낮음         없음

MEO(~10,000 km) → 내/외부 대 사이 "슬롯" 구간 활용
방사선 강화 부품(Rad-Hard) 필수
```

### MEO 대표 시스템

| 시스템 | 고도 | 용도 |
|--------|------|------|
| GPS (미국) | 20,200 km | 항법 |
| GLONASS (러시아) | 19,100 km | 항법 |
| Galileo (EU) | 23,222 km | 항법 |
| BeiDou MEO (중국) | 21,528 km | 항법 |
| O3b / SES | 8,063 km | 브로드밴드 통신 |

## 3. GEO (Geostationary Earth Orbit)

### 궤도 역학

```
GEO 조건: 공전 주기 = 지구 자전 주기 = 23시간 56분 4초

정지 궤도 고도 계산:
  T = 2π × √(a³/GM)
  a = ∛(GM × T²/4π²)
    = ∛(3.986×10¹⁴ × (86164)² / 4π²)
    ≈ 42,164 km (지구 중심 기준)
    = 42,164 - 6,371 ≈ 35,793 km (지표 기준)

적도 상공 고정 → 위도 ±81° 이내 커버
극지방 커버 불가 (仰角 < 5°)
```

### GEO 링크 버짓 예시

```python
def link_budget(eirp_dBW, fspl_dB, gt_dB_K, bandwidth_MHz):
    """
    EIRP   : 송신 등가등방복사전력 (dBW)
    FSPL   : 자유공간 경로손실 (dB)
    G/T    : 수신 시스템 성능 지수 (dB/K)
    BW     : 수신 대역폭 (MHz)
    """
    k = -228.6  # 볼츠만 상수 (dBW/K/Hz)
    bw_dBHz = 10 * math.log10(bandwidth_MHz * 1e6)

    cn0 = eirp_dBW - fspl_dB + gt_dB_K - k  # C/N0 (dBHz)
    cn  = cn0 - bw_dBHz                       # C/N  (dB)
    return cn0, cn

import math
cn0, cn = link_budget(
    eirp_dBW = 50,      # 위성 EIRP 50 dBW
    fspl_dB  = 209.6,   # GEO Ka-band 26.5 GHz
    gt_dB_K  = 18.0,    # 지상국 G/T 18 dB/K
    bandwidth_MHz = 36  # 36 MHz 트랜스폰더
)
print(f"C/N₀: {cn0:.1f} dBHz")
print(f"C/N : {cn:.1f} dB")
```

### GEO 전파 지연의 영향

```
GEO RTT ≈ 540 ms

TCP 성능 영향:
  - 윈도우 크기 W = 64 KB 기준:
    처리량 = W / RTT = 65,536 × 8 / 0.54 ≈ 970 kbps
    → 1 Mbps도 안 되는 TCP 성능 (물리 대역폭이 수백 Mbps여도)

  해결책:
  ① PEP (Performance Enhancing Proxy): TCP 분할
  ② 대용량 TCP 윈도우 (TCP SACK + 윈도우 스케일링)
  ③ QUIC / UDP 기반 프로토콜로 전환
  ④ DVB-S2X: 전용 위성 전송 표준 (ACM, VCM)
```

## 군용 통신 관점 비교

### 생존성 (Survivability)

```
GEO 단일 위성:
  ┌──────────┐
  │ GEO SAT  │ ← 단일 장애점 (Single Point of Failure)
  └────┬─────┘   공격/재밍 시 전체 서비스 중단
       │
  [전 지역 단말]

LEO 군집:
  ★  ★  ★  ★  ★  ← 수백~수천 기 분산
     ★  ★  ★  ★
  어느 위성이 피격되어도 나머지로 서비스 지속
  → 군사적 생존성 현저히 높음
```

### 항재밍 관점

| 항목 | LEO | GEO |
|------|-----|-----|
| 수신 신호 강도 | 강함 (경로손실 적음) | 약함 |
| 재밍 대응 | 높은 링크 마진 활용 | 마진 부족 |
| 스팟빔 크기 | 소형 (수십~수백 km) | 광역 (수천 km) |
| 재밍 지향성 | 좁은 빔 → 높은 G/T 가능 | 광대역 재밍에 취약 |
| 주파수 도약 | 짧은 가시 시간 활용 | 장시간 고정 → 추적 용이 |

### 전술 통신 시나리오

```
시나리오: 해상 작전 (수상함 + 잠수함 + 항공기)

    [LEO 위성 군집]
          ↕ (저지연 4 ms)
    ┌─────────────────────────────────┐
    │  수상함 A ──── 수상함 B         │
    │      │              │           │
    │  잠수함 (부상 시)   │           │
    │                  항공기 (UAV)   │
    └─────────────────────────────────┘

LEO 장점:
  - 저지연 → 실시간 전술 데이터링크 가능
  - 다중 경로 → 재밍/피격 대응
  - 스팟빔 → 집중 대역폭 제공
  - 극지방 포함 전 해역 커버
```

## 궤도 선택 의사결정 가이드

```
통신 시스템 설계 시 궤도 선택 흐름:

서비스 요구사항
    │
    ├── 지연 < 50 ms 필수?
    │       └── YES → LEO
    │
    ├── 극지방 커버 필요?
    │       └── YES → LEO 또는 HEO (몰니야 궤도)
    │
    ├── 위성 수 최소화?
    │       └── YES → GEO (3기로 이론상 글로벌)
    │
    ├── 항법/측위 목적?
    │       └── YES → MEO (GPS/GNSS 표준)
    │
    ├── 군용 생존성 최우선?
    │       └── YES → LEO 군집
    │
    └── 고정 지상국, 단순 운용?
            └── YES → GEO
```

│                                              │

│         │                                    │
│  MEO (10,000 km)  ─── 항법 + 중용량 백홀    │
│         │                                    │
│  LEO (550 km)     ─── 저지연 브로드밴드      │
│         │                                    │
│  HAPS (20 km)     ─── 도심/재난 구간 보완    │
│         │                                    │
│  지상 5G/6G RAN   ─── 최종 액세스            │
└──────────────────────────────────────────────┘

→ 3GPP Rel-19 / 6G: 이종 궤도 핸드오버 표준화 예정
