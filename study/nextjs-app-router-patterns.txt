## The Rendering Mental Model

The App Router introduces a fundamental split: **Server Components** (RSC) and **Client Components**. Understanding where code runs is the foundation of everything else.

```
┌─────────────────────────────────────────────────────┐
│               Server Components (default)            │
│  ✓ Access DB/filesystem directly                    │
│  ✓ Zero JS shipped to browser                       │
│  ✓ Can import server-only libraries                 │
│  ✗ No useState, useEffect, event handlers           │
│  ✗ No browser APIs                                  │
├─────────────────────────────────────────────────────┤
│         Client Components ('use client')             │
│  ✓ useState, useEffect, useCallback                 │
│  ✓ Event handlers, animations                       │
│  ✓ Browser APIs (window, localStorage)              │
│  ✗ Cannot import server-only code                   │
└─────────────────────────────────────────────────────┘
```

The key insight: **push interactivity to the leaves**. Keep page/layout/data-fetching logic as RSC, add `'use client'` only at the interactive leaf components.

## Layouts and the Rendering Tree

Layouts persist across navigations without unmounting — they hold their state and don't re-render.

```jsx
// app/layout.jsx — root layout, always present
export default function RootLayout({ children }) {
  return (
    <html lang="en">
      <body>
        <Nav />         {/* Renders once, stays mounted */}
        <main>{children}</main>
        <Footer />
      </body>
    </html>
  )
}

// app/dashboard/layout.jsx — nested layout
export default function DashboardLayout({ children }) {
  return (
    <div className="dashboard">
      <Sidebar />       {/* Stays mounted during dashboard navigation */}
      <div>{children}</div>
    </div>
  )
}
```

### Parallel Routes — Multiple Slots

Render multiple pages in the same layout simultaneously:

```
app/
  dashboard/
    layout.jsx        ← receives @analytics and @team as props
    @analytics/
      page.jsx
    @team/
      page.jsx
    page.jsx          ← default slot
```

```jsx
// app/dashboard/layout.jsx
export default function DashboardLayout({ children, analytics, team }) {
  return (
    <div>
      {children}
      <div className="sidebar">
        {analytics}
        {team}
      </div>
    </div>
  )
}
```

## Data Fetching Patterns

### Fetch in Server Components (recommended)

```jsx
// app/products/page.jsx — Server Component
async function getProducts() {
  const res = await fetch('https://api.example.com/products', {
    next: { revalidate: 3600 }  // Cache for 1 hour
  })
  if (!res.ok) throw new Error('Failed to fetch')
  return res.json()
}

export default async function ProductsPage() {
  const products = await getProducts()  // Direct async/await
  return <ProductList products={products} />
}
```

### Caching Strategies

| Strategy | Code | Use Case |
|---------|------|----------|
| Static (build-time) | `fetch(url, { cache: 'force-cache' })` | Blog posts, docs |
| Time-based revalidation | `fetch(url, { next: { revalidate: 60 } })` | Product listings |
| On-demand revalidation | `revalidatePath('/products')` | After CMS update |
| Dynamic (no cache) | `fetch(url, { cache: 'no-store' })` | User-specific data |

### Parallel Data Fetching

```jsx
// ❌ Sequential — each waits for previous (waterfall)
const user = await getUser(id)
const posts = await getPosts(id)

// ✅ Parallel — both start simultaneously
const [user, posts] = await Promise.all([
  getUser(id),
  getPosts(id)
])
```

### Streaming with Suspense

```jsx
// app/dashboard/page.jsx
import { Suspense } from 'react'

export default function Dashboard() {
  return (
    <main>
      <h1>Dashboard</h1>
      {/* Stats loads immediately */}
      <StaticStats />

      {/* Slow component streams in when ready */}
      <Suspense fallback={<ChartSkeleton />}>
        <RevenueChart />   {/* async Server Component */}
      </Suspense>

      <Suspense fallback={<TableSkeleton />}>
        <RecentOrders />   {/* async Server Component */}
      </Suspense>
    </main>
  )
}
```

## Server Actions — Mutating Data

Server Actions are async functions that run on the server, called from Client Components. They replace API routes for form submissions and mutations.

```jsx
// app/actions.js
'use server'

import { revalidatePath } from 'next/cache'
import { db } from '@/lib/db'

export async function createPost(formData) {
  const title = formData.get('title')
  const content = formData.get('content')

  await db.post.create({ data: { title, content } })
  revalidatePath('/posts')  // Bust cache, trigger re-render
}
```

```jsx
// app/posts/new/page.jsx — Server Component form
import { createPost } from '@/app/actions'

export default function NewPostPage() {
  return (
    <form action={createPost}>
      <input name="title" required />
      <textarea name="content" required />
      <button type="submit">Create</button>
    </form>
  )
}
```

For client-side forms with pending state:

```jsx
'use client'
import { useFormStatus } from 'react-dom'
import { createPost } from '@/app/actions'

function SubmitButton() {
  const { pending } = useFormStatus()
  return <button disabled={pending}>{pending ? 'Creating...' : 'Create'}</button>
}

export function NewPostForm() {
  return (
    <form action={createPost}>
      <input name="title" />
      <SubmitButton />
    </form>
  )
}
```

## Route Handlers vs Server Actions

| | Route Handler | Server Action |
|--|--------------|--------------|
| Location | `app/api/*/route.js` | Any file with `'use server'` |
| Trigger | HTTP request (external or client) | Form action / `startTransition` |
| Response | Full HTTP response | Return value |
| Use when | External API, webhooks, file downloads | Form mutations, internal data writes |

```js
// app/api/webhook/route.js — Route Handler
export async function POST(request) {
  const payload = await request.json()
  await processWebhook(payload)
  return Response.json({ ok: true })
}
```

## Error Handling

```
app/
  error.jsx         ← catches errors in page.jsx children
  global-error.jsx  ← catches errors in root layout
  not-found.jsx     ← rendered by notFound()
  loading.jsx       ← automatic Suspense wrapper
```

```jsx
// app/error.jsx — must be a Client Component
'use client'

export default function Error({ error, reset }) {
  return (
    <div>
      <h2>Something went wrong</h2>
      <p>{error.message}</p>
      <button onClick={reset}>Try again</button>
    </div>
  )
}
```

## Common Patterns Checklist

- Default to Server Components — add `'use client'` only when needed
- Use `loading.jsx` for skeleton UIs (auto-wraps page in Suspense)
- Use `error.jsx` for error boundaries per route segment
- Colocate server actions near the forms that use them
- Use `generateStaticParams` for static generation of dynamic routes
