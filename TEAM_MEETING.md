## 팀 토론 요약

### 핵심 쟁점 1 — 현재 구조로 갈 것인가, 새로 시작할 것인가

**민준 (PM):**
> 지금 구조는 포트폴리오라기보다 개인 노트 앱에 가깝다. 홈, 프로젝트, 소개가 없으면 포트폴리오로 기능하지 못한다.

**지훈 (리서쳐):**
> 오히려 `Study` 페이지가 강점이다. 꾸준히 공부하는 개발자임을 증명하는 가장 직접적인 수단. 이걸 중심으로 확장하는 것이 맞다.

**태양 (리뷰어):**
> 파일 내용을 확인하지 못한 상태에서 "기존 코드 활용"을 전제하면 안 된다. 처음부터 새로 짜고 마크다운/i18n 로직만 이식하는 게 안전하다.

**→ 합의:** 신규 프로젝트로 시작. 기존 `markdown.js`와 `translations.js` 로직만 가져온다.

---

### 핵심 쟁점 2 — 프레임워크: Vite+React vs Next.js

**민준:**
> SEO 생각하면 Next.js가 맞다. 포트폴리오는 검색 노출이 중요하다.

**서연 (개발자):**
> 포트폴리오 규모에서 Next.js는 오버스펙. Vite + React Router로도 `react-helmet`으로 SEO 해결 가능. 실제 유입은 GitHub/LinkedIn 링크가 더 많다.

**→ 미결:** 앨런이 결정 필요.  
- SEO + 장기 블로그 확장 고려 → **Next.js**  
- 빠른 개발, 가벼운 구성 → **Vite + React**

---

### 핵심 쟁점 3 — 포트폴리오의 정체성

**수아 (디자이너):**
> 첫 화면 3초 안에 "누구인지" 안 보이면 이탈. Study만 있으면 블로그인지 포트폴리오인지 구분 불가. Hero 섹션은 필수.

**태양:**
> 프로젝트 목록보다 "어떤 문제를 어떻게 해결했는가"라는 스토리가 중요하다. Study 페이지가 그 성장 증거 역할을 할 수 있다.

**지훈:**
> GitHub API 연동으로 "지금도 코딩하는 사람"이라는 신호를 줄 수 있다.

**→ 합의:** Study 페이지를 차별점으로 키운다. Hero는 필수, 프로젝트는 스토리 중심으로 작성.

### 민준 (PM · 아키텍트) — 사이트맵 & 우선순위

**권장 사이트맵:**
```
/                   → 홈 (Hero + 기술스택 + 프로젝트 요약)
/about              → 소개 (자기소개, 경력, 기술스택 상세)
/projects           → 프로젝트 목록
/projects/:id       → 프로젝트 상세 (문제 → 해결 → 결과)
/study              → 학습 노트 목록
/study/:slug        → 노트 상세 (마크다운 렌더링)
/contact            → 연락처
```

**MVP 우선순위:**
| 순위 | 페이지/기능 | 이유 |
|------|------------|------|
| P0 | Home Hero | 첫인상 결정 |
| P0 | Projects | 채용담당자 최우선 확인 항목 |
| P0 | About + 기술스택 | 신뢰도 |
| P1 | Study 페이지 | 기존 기반 활용, 차별점 |
| P1 | 반응형 + 다크모드 | 완성도 기준선 |
| P2 | i18n 한/영 전환 | 글로벌 접근성 |
| P2 | Contact | 채용 연결 |
| P3 | GitHub API 연동 | 활동성 어필 |

### 지훈 (리서쳐) — 콘텐츠 전략

**Study 페이지를 핵심 자산으로:**
- 기술 블로그 겸용 → SEO 자연 유입 창구
- 카테고리 분류 (JavaScript, React, CS 기초 등)
- 읽기 시간 표시
- 검색 기능

**프로젝트 스토리텔링 구조:**
```
각 프로젝트 카드:
  - 문제: 어떤 불편함/필요에서 시작했는가
  - 해결: 어떤 기술적 결정을 내렸는가
  - 결과: 수치 또는 경험으로 표현된 성과
  - 기술 스택 태그
  - GitHub 링크 / 라이브 데모
```

**추가 콘텐츠 아이디어:**
- GitHub API: 최근 커밋, 기여 잔디, 주요 레포
- 타임라인: 학습/커리어 히스토리
- 기술 레벨 표시 (단순 나열 X, 숙련도 구분)
