5G System Architecture Analysis

May 15, 2026·9 min read

Overview

5G NR (New Radio) is the next-generation mobile communication system whose first standard was finalized in 3GPP Rel-15 (2018). Compared to LTE, it targets the following goals:

| Metric | LTE (4G) | 5G NR Target | |--------|----------|--------------| | Peak Downlink Speed | ~1 Gbps | 20 Gbps | | User Experience Speed | ~10 Mbps | 100 Mbps | | Latency (E2E) | ~10 ms | < 1 ms | | Connection Density | 10⁵ /km² | 10⁶ /km² | | Mobility | 350 km/h | 500 km/h | | Energy Efficiency | Baseline | 100x improvement |

1. 5G Core Network (5GC)

The 5GC adopts a Service-Based Architecture (SBA) compared to LTE's EPC (Evolved Packet Core). All functions are separated as microservices.

Key Network Functions (NF)

┌────────────────────────────────────────────────────────────────┐
│                    Service Based Interface (SBI)               │
│                    (HTTP/2 + JSON/Protobuf)                    │
├────────┬────────┬────────┬────────┬────────┬────────┬─────────┤
│  AMF   │  SMF   │  UPF   │  PCF   │  UDM   │  NRF   │  AUSF  │
│Mobility│Session │Packet  │Policy  │Subscriber│Function│ Auth  │
│Mgmt    │Mgmt    │Forward │Control │ Data   │Registry│        │
└────────┴────────┴────────┴────────┴────────┴────────┴─────────┘

| NF | Role | LTE Equivalent | |----|------|----------------| | AMF | Mobility/Access/Security Management | MME | | SMF | PDU Session Management, IP Allocation | SGW-C + PGW-C | | UPF | User Plane Packet Forwarding | SGW-U + PGW-U | | PCF | Policy/QoS Control | PCRF | | UDM | Subscriber Profile Storage | HSS | | NRF | NF Registration/Discovery (Service Mesh) | None (new) | | AUSF | Authentication Server | Part of HSS | | NSSF | Network Slice Selection | None (new) |

Control Plane / User Plane Separation (CUPS)

Control Plane:    UE ─── AMF ─── SMF ─── PCF/UDM
                              (signaling path)

User Plane:       UE ─── UPF ─── DN (Internet/IMS)
                              (data path)

→ UPF can be placed at the edge (MEC) → ultra-low latency services

3. Physical Layer (PHY) Key Characteristics

NR Numerology

5G NR allows flexible subcarrier spacing configuration.

| μ | SCS (kHz) | Symbols/Slot | Slot Duration | Primary Use | |---|-----------|--------------|---------------|-------------| | 0 | 15 | 14 | 1 ms | FR1, LTE compatible | | 1 | 30 | 14 | 0.5 ms | FR1 general NR | | 2 | 60 | 14 | 0.25 ms | FR1/FR2 | | 3 | 120 | 14 | 0.125 ms | FR2 (mmWave) | | 4 | 240 | 14 | 0.0625 ms | FR2 reference signals |

def slot_duration_us(mu):
    """Calculate NR slot duration"""
    scs_kHz = 15 * (2 ** mu)        # subcarrier spacing
    slot_per_subframe = 2 ** mu      # slots per subframe
    slot_duration = 1000 / slot_per_subframe  # in ms
    return scs_kHz, slot_duration

for mu in range(5):
    scs, dur = slot_duration_us(mu)
    print(f"μ={mu}: SCS={scs:4d} kHz, slot={dur:.4f} ms")

# μ=0: SCS=  15 kHz, slot=1.0000 ms
# μ=1: SCS=  30 kHz, slot=0.5000 ms
# μ=2: SCS=  60 kHz, slot=0.2500 ms
# μ=3: SCS= 120 kHz, slot=0.1250 ms
# μ=4: SCS= 240 kHz, slot=0.0625 ms

Frequency Ranges (FR)

FR1 (Sub-6 GHz): 410 MHz ~ 7.125 GHz
  - Can reuse existing LTE bands
  - Favorable for wide-area coverage
  - Max bandwidth: 100 MHz

FR2 (mmWave): 24.25 GHz ~ 52.6 GHz
  - Ultra-wideband (up to 400 MHz / carrier)
  - Capable of ultra-high speeds (tens of Gbps)
  - High attenuation and poor diffraction → beamforming required

Beamforming Types

Analog Beamforming:
  [ADC/DAC]─[Phase Shifter]─[Antenna]
  → Simple, low cost, single beam only

Digital Beamforming:
  Independent ADC/DAC per antenna
  [Digital Processing]─[ADC/DAC]─[Antenna]
  → Multiple simultaneous beams, flexible, high cost

Hybrid Beamforming (mainstream in 5G NR):
  [Digital Precoder]─[Analog Phase Shifter]─[Antenna Subarray]
  → Compromise: few digital chains + many analog phase shifters

import numpy as np

def hybrid_beamforming_weight(N_antenna, N_beam, angles_deg):
    """
    N_antenna: number of antennas
    N_beam:    number of simultaneous beams
    angles_deg: direction of each beam (degrees)
    """
    d = 0.5  # antenna spacing (in wavelengths)
    W = np.zeros((N_antenna, N_beam), dtype=complex)

    for i, theta in enumerate(angles_deg):
        theta_rad = np.deg2rad(theta)
        # analog steering vector
        steering = np.exp(1j * 2 * np.pi * d *
                          np.arange(N_antenna) * np.sin(theta_rad))
        W[:, i] = steering / np.sqrt(N_antenna)

    return W

# 8 antennas, 3 beams (-30°, 0°, +30°)
W = hybrid_beamforming_weight(8, 3, [-30, 0, 30])
print(f"Beamforming weight matrix shape: {W.shape}")  # (8, 3)

5. Network Slicing

A single physical infrastructure is logically separated to provide customized networks per service.

Physical Infrastructure (shared)
┌───────────────────────────────────────────────────┐
│  Compute   Storage   Network   RAN   Core          │
└───────────────────────────────────────────────────┘
                    ↕ Virtualization (NFV/SDN)
┌──────────────┬──────────────┬──────────────────────┐
│   Slice 1    │   Slice 2    │     Slice 3          │
│   eMBB       │   URLLC      │     mMTC             │
│  (Broadband) │ (Ultra-Low   │  (Massive IoT)       │
│              │  Latency)    │                       │
│ ↑ 20 Gbps   │ ↓ &lt;1 ms     │ ↑ 10⁶ dev/km²        │
│  Smartphone  │  Autonomous  │  Sensors/Meters      │
│  AR/VR       │  Driving     │  Smart City          │
└──────────────┴──────────────┴──────────────────────┘

S-NSSAI (Single-Network Slice Selection Assistance Information)

S-NSSAI = SST (8-bit) + SD (24-bit, optional)

Standard SST values:
  SST=1: eMBB (Enhanced Mobile Broadband)
  SST=2: URLLC (Ultra-Reliable Low Latency)
  SST=3: MIoT (Massive IoT)
  SST=4: V2X (Vehicle-to-Everything)

8. Modem SW Development Perspective

Layer-by-Layer Development Points

PHY Layer:
  - LDPC/Polar channel coding implementation
  - HARQ process management
  - Beam management (Beam Measurement/Report/Switch)
  - UCI (Uplink Control Information) encoding

MAC Layer:
  - Scheduler interface (UL/DL grant processing)
  - BSR (Buffer Status Report) generation
  - SR (Scheduling Request) triggering
  - Timing Advance management

RLC Layer:
  - AM mode: ARQ retransmission window management
  - Segmentation/Reassembly processing
  - PDCP duplication support

PDCP Layer:
  - ROHC header compression (IP/UDP/RTP)
  - Encryption/Decryption (AES-CTR, SNOW 3G)
  - Integrity verification (Control Plane)
  - Out-of-order delivery processing
