## Overview

The LTE (Long Term Evolution) Radio Access Layer is composed of multiple sublayers. This document summarizes the role and interaction of each layer.

## Layer Structure

```
┌───────────────────────┐
│      Application      │
├───────────────────────┤
│   IP / Non-IP (NAS)   │
├───────────────────────┤
│         PDCP          │  ← Header compression, encryption
├───────────────────────┤
│          RLC          │  ← ARQ, segmentation/reassembly
├───────────────────────┤
│          MAC          │  ← Scheduling, HARQ
├───────────────────────┤
│          PHY          │  ← OFDMA/SC-FDMA, modulation
└───────────────────────┘
```

## PHY Layer (Physical Layer)

The PHY layer handles actual radio signal transmission and reception.

- **Downlink**: OFDMA (Orthogonal Frequency Division Multiple Access)
- **Uplink**: SC-FDMA (Single Carrier FDMA)
- **Modulation schemes**: QPSK, 16QAM, 64QAM
- **Channel coding**: Turbo Code

### Resource Block Structure

| Item | Value |
|------|-------|
| Subcarrier spacing | 15 kHz |
| Subcarriers per RB | 12 |
| Symbols per slot | 7 (Normal CP) |
| TTI | 1ms (1 subframe) |

## MAC Layer (Medium Access Control)

### HARQ (Hybrid ARQ)

HARQ combines ARQ and FEC for error correction.

```c
/* HARQ Process state example (pseudocode) */
typedef struct {
    uint8_t process_id;
    bool    ack_received;
    uint8_t retx_count;
    uint8_t rv;           /* Redundancy Version */
} harq_process_t;

void harq_feedback_handler(harq_process_t *proc, bool ack) {
    if (ack) {
        proc->ack_received = true;
        proc->retx_count = 0;
    } else {
        proc->retx_count++;
        proc->rv = (proc->rv + 1) % 4;  /* RV cycles: 0, 2, 3, 1 */
    }
}
```

## RLC Layer (Radio Link Control)

RLC supports three operating modes:

- **TM (Transparent Mode)**: Transparent delivery without processing
- **UM (Unacknowledged Mode)**: Unidirectional, no retransmission
- **AM (Acknowledged Mode)**: Bidirectional, ARQ retransmission

## PDCP Layer (Packet Data Convergence Protocol)

- Header compression: ROHC (Robust Header Compression)
- Encryption: AES-128 (EEA2), Snow 3G (EEA1)
- Integrity protection: HMAC-SHA-256 (EIA2)

## Summary

Each layer of the LTE protocol stack performs an independent role while exchanging data through interfaces. Maintaining this layer separation principle during modem SW development is critical for maintainability.
