Next.js 14's App Router is a new routing system designed around React Server Components (RSC). It differs conceptually from the `pages/` directory approach, and misunderstanding the correct patterns leads to performance degradation or unexpected behavior.

## Server Components vs Client Components

The core concept of App Router. All components are **Server Components** by default.

### Server Component (default)

```tsx
// app/users/page.tsx — runs only on the server
async function UsersPage() {
  // Direct DB queries and API calls are possible
  const users = await db.users.findMany();
  
  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

Server components cannot use `useState` or `useEffect`, nor event handlers. Instead, they can directly access databases, read the file system, and use server-side environment variables.

### Client Component

```tsx
'use client'; // this directive is required for client components

import { useState } from 'react';

export function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(c => c + 1)}>
      Clicks: {count}
    </button>
  );
}
```

### Component Composition Pattern

```tsx
// app/dashboard/page.tsx — Server Component
import { ClientChart } from './ClientChart';

async function DashboardPage() {
  const data = await fetchDashboardData(); // fetch data on server
  
  return (
    <div>
      <h1>Dashboard</h1>
      <ClientChart data={data} /> {/* pass data to client component */}
    </div>
  );
}
```

Core principle: **fetch data on the server, handle interactions on the client**.

## File System Based Routing

```
app/
├── layout.tsx          # Root layout (required)
├── page.tsx            # / route
├── loading.tsx         # Loading UI
├── error.tsx           # Error UI
├── not-found.tsx       # 404 UI
├── (marketing)/        # Route group — not included in URL
│   ├── about/page.tsx  # /about
│   └── blog/page.tsx   # /blog
├── dashboard/
│   ├── layout.tsx      # Dashboard layout
│   └── page.tsx        # /dashboard
└── api/
    └── users/
        └── route.ts    # /api/users API route
```

### Dynamic Routes

```tsx
// app/users/[id]/page.tsx
interface Props {
  params: { id: string };
  searchParams: { tab?: string };
}

export default async function UserPage({ params, searchParams }: Props) {
  const user = await getUser(params.id);
  const activeTab = searchParams.tab ?? 'profile';
  
  return <UserDetail user={user} activeTab={activeTab} />;
}

// Pre-define paths for static generation
export async function generateStaticParams() {
  const users = await getAllUsers();
  return users.map(u => ({ id: u.id }));
}
```

## Data Fetching Patterns

### Parallel Fetching

```tsx
async function UserDashboard({ userId }: { userId: string }) {
  // Sequential execution (bad pattern)
  // const user = await getUser(userId);        // wait
  // const posts = await getUserPosts(userId);  // wait again
  
  // Parallel execution (good pattern)
  const [user, posts, stats] = await Promise.all([
    getUser(userId),
    getUserPosts(userId),
    getUserStats(userId)
  ]);
  
  return <Dashboard user={user} posts={posts} stats={stats} />;
}
```

### Streaming with Suspense

```tsx
import { Suspense } from 'react';

async function SlowDataComponent() {
  const data = await fetchSlowData(); // request that takes 5 seconds
  return <div>{data.value}</div>;
}

export default function Page() {
  return (
    <div>
      <h1>Fast content</h1>
      <Suspense fallback={<div>Loading slow data...</div>}>
        <SlowDataComponent />
      </Suspense>
    </div>
  );
}
```

## API Route Handlers

```typescript
// app/api/users/route.ts
import { NextRequest, NextResponse } from 'next/server';

export async function GET(request: NextRequest) {
  const searchParams = request.nextUrl.searchParams;
  const page = Number(searchParams.get('page') ?? '1');
  
  const users = await db.users.findMany({
    skip: (page - 1) * 10,
    take: 10
  });
  
  return NextResponse.json({ users, page });
}

export async function POST(request: NextRequest) {
  const body = await request.json();
  
  // Validation
  const parsed = UserCreateSchema.safeParse(body);
  if (!parsed.success) {
    return NextResponse.json(
      { error: parsed.error.flatten() },
      { status: 400 }
    );
  }
  
  const user = await db.users.create({ data: parsed.data });
  return NextResponse.json(user, { status: 201 });
}
```

## Server Actions

Used for form submissions and data mutations. Directly calls server logic without API routes:

```tsx
// app/profile/actions.ts
'use server';

import { revalidatePath } from 'next/cache';

export async function updateProfile(formData: FormData) {
  const name = formData.get('name') as string;
  const bio = formData.get('bio') as string;
  
  await db.users.update({
    where: { id: getCurrentUserId() },
    data: { name, bio }
  });
  
  revalidatePath('/profile'); // invalidate cache
}
```

```tsx
// app/profile/page.tsx
import { updateProfile } from './actions';

export default function ProfilePage() {
  return (
    <form action={updateProfile}>
      <input name="name" />
      <textarea name="bio" />
      <button type="submit">Save</button>
    </form>
  );
}
```

## Cache Control

```typescript
// Default: automatic cache
const data = await fetch('/api/data');

// No cache (always fresh)
const data = await fetch('/api/data', { cache: 'no-store' });

// Revalidate every 60 seconds
const data = await fetch('/api/data', { next: { revalidate: 60 } });

// Tag-based invalidation
const data = await fetch('/api/posts', { next: { tags: ['posts'] } });

// Manual invalidation (from Server Action)
import { revalidateTag } from 'next/cache';
revalidateTag('posts');
```

## Middleware

```typescript
// middleware.ts (at root)
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  const token = request.cookies.get('token');
  
  if (!token && request.nextUrl.pathname.startsWith('/dashboard')) {
    return NextResponse.redirect(new URL('/login', request.url));
  }
  
  return NextResponse.next();
}

export const config = {
  matcher: ['/dashboard/:path*', '/api/protected/:path*']
};
```

## Migration Checklist: pages/ to app/

- [ ] `getServerSideProps` → replace with `async` server components
- [ ] `getStaticProps` → `fetch` with `cache: 'force-cache'`
- [ ] `getStaticPaths` → `generateStaticParams`
- [ ] `useRouter` (pages) → `useRouter` (next/navigation)
- [ ] Remove `<a>` inside `next/link` (unnecessary in App Router)
- [ ] `_app.tsx` logic → move to `app/layout.tsx`
- [ ] API Routes → convert to Route Handlers

The App Router has a steep learning curve, but using Server Components by default dramatically reduces client bundle size and simplifies data fetching. The async server component pattern is more intuitive than the `getServerSideProps` approach from `pages/` and is quickly becoming the industry standard.
